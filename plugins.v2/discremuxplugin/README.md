# 蓝光原盘重封装

MoviePilot v2 插件：基于整理历史或下载器拦截，将蓝光原盘重封装为 MKV。

## 依赖

- 需要 `ffmpeg` 和 `ffprobe` 命令可用。
- FFmpeg 必须编译启用 `bluray` 协议（通常需要 `libbluray` 支持）。可用 `ffmpeg -protocols | grep bluray` 确认。

## 工作模式

### 整理历史重封装

读取 MoviePilot 最近整理历史，找到已经入库到媒体库的蓝光原盘目录，使用 FFmpeg 的 `bluray:` 协议读取媒体库中的 BDMV，自动选择时长最长的播放列表重封装为 MKV，并输出到当前媒体库条目目录。

成功后可按配置：

- 为旧 BDMV 创建 `.ignore`。
- 删除旧 `BDMV` / `CERTIFICATE`。
- 删除下载源。
- 刷新对应媒体库。

该模式不会删除或篡改 MoviePilot 整理记录。

### 监听下载器重封装

在 MoviePilot 整理下载器中的蓝光原盘任务前拦截原整理，先在下载目录重封装 MKV，再用原下载历史中的媒体 ID 发起 MKV 整理。

该模式下，MoviePilot 原始原盘整理记录会保留为失败；插件详情页会记录接管后的重封装和后续整理结果。

成功后可按配置：

- 删除下载目录中的原盘源目录。
- 调用 MoviePilot 整理重封装后的 MKV。
- 刷新对应媒体库。

## 配置说明

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| 跳过阈值 | 目标 MKV 已存在且大于该大小时跳过 FFmpeg 重封装 | `5GB` |
| 只处理电影 | 仅处理电影类型记录或下载历史 | 开启 |
| 成功后删除下载源 | 成功后删除对应下载源；整理历史模式删除历史源文件，监听模式删除被拦截的原盘目录 | 关闭 |
| 尝试刷新对应媒体库 | 调用媒体服务器刷新能力 | 开启 |
| 启用定时重封装 | 开启整理历史重封装定时服务 | 关闭 |
| 定时运行策略 | 整理历史重封装 Cron 表达式 | `0 3 * * *` |
| 查询最近整理记录天数 | 整理历史模式查询最近 N 天记录 | `7` |
| 旧媒体库 BDMV 处理方式 | 整理历史模式中创建 `.ignore` 或删除旧 `BDMV` / `CERTIFICATE` | 创建 `.ignore` |
| 启用下载器完成后拦截重封装 | 开启监听下载器重封装 | 关闭 |
| 重封装后发起整理 | 监听模式中重封装完成后调用 MoviePilot 整理 MKV | 开启 |

## 跳过条件

- 目标 MKV 已存在且大小大于配置阈值。
- 整理历史模式中，旧媒体库 BDMV 内已存在 `.ignore`。
- 插件数据中已记录对应处理流水。
- 原盘目录不存在，或缺少 `index.bdmv` / `MovieObject.bdmv` 标志文件。

## 插件详情页

- 显示最近处理流水，包括模式、状态、来源、输出、下载源处理、后处理和时间。
- “清空已处理历史”会清空插件处理流水，允许重新评估历史记录。
